## 어떤 주제로 학습을 했나요?

**상혁**
```
JVM의 메모리 구조와 각 영역 별 OOM이 발생하는 원인
```

**수영**
```
JPA, DB

처음에는 어떤 것을 할지 잘 모르겠어서, 워키나 회사 등 개발을 하다가 궁금해 지는 것들에 대해서 정리를 하는 식으로 진행했고
지금은 DB 트랜잭션이라는, 공부해보고 싶었던 주제를 잡아서 다음 주는 이걸로 좀 길게 가져가지 않을까 생각됩니다.
```

## 좋았던 점 / 잘 한 것 같은 점은 무엇인가요?

**상혁**

```
처음에는 단순히 `OOM 이 터졌는데, 어떻게 heapdump 를 뜰 수 있는거지?` 가 궁금해서 해당 부분만 찾아보려고 했었는데,
하다보니 JVM 이나 메모리 구조 영역까지 범위를 넓히게 된 점이 (덕분에 아직까지 하고 있지만) 좋았습니다.

사실 아직은 약간 눈치가 보이긴 하지만,
바쁜 날에는 `아주 조금이라도 커밋하면 된다` 는 규칙 덕분에 1일 1커밋을 최소한 놓치지는 않았던 것 같아서 좋았습니다.
```

**수영**

```
궁금한 게 생겼을 때 그냥 넘어가거나 적당히 찾아보고 지나가는게 아니라 여기에 정리하고 기록할 수 있다는 점이 좋았습니다.
양이 정해져 있는게 아니라서 부담도 없었고, 하루에 그냥 본인이 생각하는 만큼만 편하게 정리하면 되어서 좋았습니다.
```

## 개선이 필요한 부분은 무엇인가요?

**상혁**
```
지금보다는 하루에 커밋하는 양을 조금 더 가져가면 좋을 것 같습니다.

바쁘다는 핑계로 하루에 시간을 2-30분 정도씩 밖에 투자하지 못했고,
그 시간에 정리까지 해야하다보니 실제로 커밋하는 양이 매우 적었던 것 같습니다.

눈치 보여서가 아니라, 조금 더 빠르게 성장하기 위해서 하루하루 투자하는 시간을 (당장은 어렵더라도) 조금씩이라도 늘려나가고자 합니다.
```

**수영**
```
커밋을 두 번이나 놓쳤습니다… 한번은 잠들어서, 한번은 회의 때문에 놓쳤습니다.
첫 주를 진행하는 데에 있어서 좋지 못한 모습을 보여 많이 반성했습니다..ㅠㅠ

우선 피곤하거나 일정이 있으면 아침이나 점심 시간을 통해 짧게라고 올려놓도록 해야 할것 같고,
회의 같은게 있다면 시간적으로 여유롭다고 생각해도,
미리 쓴 부분까지만 커밋하고 추가적으로 쓰는 게 좋을 것 같다고 생각했습니다.

1일 1커밋이 목표이지 하루에 딱 한개만 해야 하는게 아니니까요…
```
