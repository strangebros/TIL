## 📌 개요
- 대용량 경로에서 `find`는 느리고 위험할 수 있음
- 파일 경로를 정확히 알고 있을 때는 `ls / cp / rm` 조합이 최선
- 유닉스 환경에서는 GNU 옵션(`cp -a`)이 동작하지 않을 수 있음
- 삭제 전 **서버 내부 백업 + 무결성 검증** 필수
- 삭제 후 확인은 `ls` 한 줄이면 충분

---

## 🧠 1. 환경 이슈: 왜 `cp -a`가 안 됐나
- `cp -a`는 **GNU coreutils 전용 옵션**
- AIX / Solaris / HP-UX 등 전통 유닉스 환경에서는 미지원
- 단일 파일 백업 목적이라면 `cp -p`로 충분

```sh
cp -p /원본/파일.hwp /backup/경로/
```

---

## 🧠 2. 파일 “존재 확인”만 할 때 (find 사용 ❌)

- 대용량 디렉토리에서 find는 전체 순회로 인해 무한 로딩처럼 보일 수 있음

- 정석

```sh
ls -l /경로/파일명.hwp
```

- 있으면 상세 정보 출력

- 없으면 No such file or directory

→ 존재 확인 용도로 find는 부적절

## 🧠 3. 서버 내부 백업 절차 (로컬 접근 불가 상황)
### 1️⃣ 백업 디렉토리 생성

```sh
mkdir -p /backup/manual_YYYYMMDD
```

### 2️⃣ 파일 복사

```sh
cp -p /경로/파일명.hwp /backup/manual_YYYYMMDD/
```

### 3️⃣ 무결성 검증 (유닉스 공통)

```sh
cksum /경로/파일명.hwp
cksum /backup/manual_YYYYMMDD/파일명.hwp
```

→ checksum 값이 동일하면 정상 백업

## 🧠 4. 삭제 명령어 호환성 (유닉스 기준)

```sh
rm -i : POSIX 표준 → 대부분의 유닉스에서 지원

rm -v : GNU 확장 → 환경에 따라 미지원 가능
```

- 안전한 삭제

```sh
rm -i /경로/파일명.hwp
```

- 파일마다 삭제 여부 확인 가능

## 🧠 5. 삭제 후 확인 방법

```sh
ls -l /경로/파일명.hwp

No such file or directory → 삭제 완료
```

- 추가 도구 불필요
